# 5ns 综合分析（2026-06-19）：流水化后的第二次 5ns，能否用 cycles 换时序+面积达标

> 分支 `baseline-v2-synthopt`。本文分析**第二次 5ns 综合**（RTL = `b56628d`，3 级流水
> `softmax_combine`；报告归档于 `synth/reports_ispatial_5.000ns_b56628d_PIPELINED/`），
> 回答：**目前能否用富余的 cycles 预算，换来更优的面积与时序，达到赛题硬约束？**

## TL;DR

**能，而且代价几乎为零。** 这次流水化把局面从「无望」变成「差一步」。再补 **1 级流水**
（把 `exp_w` 那一拍劈成 E1|E2 两拍）预计就能 5ns clean；而面积超标的 6.3% 几乎全是
**时序不收敛导致的虚胖**，闭合时序后会自动回落到 9.59M 硬限以下。cycle 代价 ≈ +2k（109,414→~111.5k，
仍只占 300k 预算的 37%）。

## 一、赛题三项指标的当前差距

| 指标 | 当前（`b56628d` @5ns） | 赛题要求 | 性质 | 状态 |
|---|---|---|---|---|
| 主频 / slack | −602.7 ps（worst path 5.58ns） | 越高越好 | **软分** | 未 clean，差一步 |
| 等效门数 | 2,125,138（10.19M µm²） | ≤ 2,000,000（9.59M µm²） | **硬限** | 超 6.3%（+12.5 万门） |
| Cycles | 109,414 | < 300,000 | **硬限** | ✅ 仅用 36.5%，余量 63% |

> 面积是**唯一会卡死提交的硬约束**；cycles 余量巨大正是「换」的本钱。

## 三、瓶颈定位：关键路径整条在 `exp_w` 那一拍（E 级）

`30_timing.rpt` Path 1（slack −602.7 ps）：

```
m_tile_q_reg[30] (running max, S_MAC 期间稳定)
  → (score_cur_in − m_tile_q) 减法
  → exp_w: 64 项 LUT 译码 (gt_246_36 比较链)
  → 插值乘法 csa_tree_exp_w_247_25_add_156_60 (Wallace CSA + 全加器链)
  → s1_w_reg[15]/D
data 5584 ps / required 4786 ps / slack −603 ps
```

- 第一次的关键路径是**整条单拍** `m_tile→exp→64 路乘→累加 = 9780 ps`；流水化已切掉乘法和累加，
  **现在只剩 `exp_w` 单函数本身就 5584 ps**。
- `exp_w` 是**纯前馈**组合锥（`m_tile_q` 在整个 MAC 窗口稳定，逐拍变化的是 `score_cur_in`），
  与 loop-carried 的累加器无关 → **可自由插入流水寄存器，不影响 II=1、不改算术**。
- retiming（已开，high）救不了：这条锥两端才有触发器、中间无富余寄存器可挪，**必须从 RTL 加一级**。

## 四、面积超标几乎全是时序虚胖（硬证据，这是「换」最关键的一环）

比对**不在关键路径上**的 DMA/AXI 子系统（各分支同一份 RTL，可直接对比）：

| 非核心区域 = top − u_flash_core − u_axi_lite_regs − u_axi_master_read | 值 | 来源 |
|---|---|---|
| 本次失败 5ns（`b56628d`，638 违例） | **3.35M µm²** | `20_area.rpt` |
| 第一次失败 5ns（`699bff9`，5650 违例） | 3.36M µm² | FAILED 归档 |
| **v2work 真实 8ns clean**（slack +1.7ps，0 违例） | **1.76M µm²** | `~/v2work/synth/reports_ispatial/20_area.rpt` |

DMA/AXI 根本不在关键路径上，却被 Genus「全程追时序」模式普涨驱动到 **~2 倍**。0 违例后 `syn_opt`
面积回收会把它降回 ~1.76M：

```
10.19M − (3.35M − 1.76M) ≈ 8.60M µm² ≈ 179 万门   →  低于 9.59M，余量 ~21 万门
保守版（非核心只回落到 2.2M）：总 ≈ 9.0M，仍达标
```

**结论：闭合时序同时解决软分(频率)和硬限(面积)，不需要先动结构去砍面积。**

## 五、行动计划

### 第一步（本次执行）：`exp_w` E-split（E1|E2），单点改动
- **E1**：`score − m_tile` + `abs` + LUT 索引 + 取 `y0/y1`（两次 64 项查表）→ 寄存
- **E2**：插值乘 `y_diff*lut_rem` + 舍入移位拼权重 → `s1_w`
- 算术逐位不变；端口 / `exp_w` 数值 / `vreq` 协议不变 → `flash_core.sv`、TB **不改**；II=1。
- cycle：每 tile-merge +1 拍 drain → 109,414 → ~111.5k（<300k）。
- 综合：effort **仍 medium**，retiming 保持开。

### 兜底阶梯（按上面的账大概率用不到）
- 时序还差：`m_tile_q<=max_comb` 的 16 路 36-bit max 也补一级；或临时 effort=high（风险：面积回涨、单跑 ~10h）。
- 面积还超（cycles 还有 63% 可烧，真·结构换面积）：
  - `dot_stream` `DOT_LANES 32→16`：点积乘法阵列腰斩，dot 周期翻倍；
  - softmax_combine 的 **64 路并行乘法（X 级）时分复用成 32/16 路**。
